= Add the App to the product store
You can find out more about applications here: xref:documentation:onecx-docs-overview:architecture/index.adoc#applications[Applications]

== Add the application to the product store

* Choose application store on the left side pannel of the OneCX shell.

* Click three dots in the top right corner of the application store and select `Add Application` from the dropdown menu.

.Locate the "Add Application" button
image::extend/add_application_manually/application_store.png[Find the "Add Application" button in the top right corner of the application store]

* Fill in the form with the following details:

.Required Application Details (example)
[.stripes-even,cols="25,~,~"]
|===
| Field | Example Value | Notes

| Name
| example
| Unique name of the application used as identifier

| Version
| xxx
| Application version

| Display Name
| example
| Name of the application displayed in the UI

| Base Path
| /my-app
| Base path for the application
|===

* Click `Save` to add the application to the product store.

.Example application details
image::extend/add_application_manually/example_application_parameters.png[Example of filled application details form]

== Add a UI component for the application

* Return to the `Application Store` and click the `Apps` button in the top right corner to change the view.

.Apps button in the application store
image::extend/add_application_manually/change_to_apps_tab.png[Find the "Apps" button in the top right corner of the application store]

* Click three dots in the top right corner of the application store and select `Create UI Component` from the dropdown menu.

.Locate the "Create UI Component" button
image::extend/add_application_manually/create_ui_component_pick.png[Find the "Create UI Component" button in the top right corner of the application store]

* Fill in the `management details` with the following information:

[NOTE]
====
let's use {module_federation_webpack_config_example_link} as a reference for `webpack.config.js` values and {module_federation_bootstrap_link} for `module.ts` bootstrapping details
====

.Required Management Details (example)
[.stripes-even,cols="25,~,~"]
|===
| Field | Example Value | Notes

| App ID
| example-ui from the {module_federation_webpack_config_example_link} example
| Unique name of the app used as identifier (module name in `webpack.config.js` config).

| App Name
| example-ui
| Name of the application displayed in the UI

| Application Name
| example
| Name of the application in the product store to which the UI component will be added.

| Version
| xxx
| Application version

|===

.Example management details
image::extend/add_application_manually/create_ui_component_management_details.png[Example of filled management details form]

* Click `Remote Module` tab in the top of the card 
* Fill in the `Remote Module` details with the following information:

.Remote Module Details (example)
[.stripes-even,cols="25,~,~"]
|===
| Field | Example Value | Notes 

| Type
| Module. See {module_federation_app_types_link} for more details.
| Type of the module federation application. 

| Technology
| WebComponent (Module). See {module_federation_app_types_link} for more details (for webpack module applications created with Angular under version 13, choose `WebComponent (Script)` technology).
| Technology used for the application. 

| Exposed Module
| ./ExampleApp from the {module_federation_webpack_config_example_link} example
| The name of the exposed module in the `webpack.config.js` file

| Remote Entry
| /mfe/example/remoteEntry.js. 
| The URL to the `remoteEntry.js` file of the application, where `app-name` depends on local app integration with `./toggle-mfe.sh` script. See xref:onecx-local-env:extend/local-mfe.adoc[Local MFE Integration]. The URL should start from `/mfe/`.

| Remote Base URL
| /mfe/example/
| The base URL for the remote module, where `app-name` depends on local app integration. The URL should start from `/mfe/`.

| Remote Name (Optional)
| example-ui from the {module_federation_webpack_config_example_link} example.
| The `uniqueName` of the remote module as defined in the `webpack.config.js` file. This is optional and only needed if the remote module has a specific name or for `WebComponent (Script)` technology.

| Tag Name (Optional)
| example-webcomponent from the {module_federation_bootstrap_link} example.
| The name of the custom element tag added in `ngDoBootstrap` method of the `module.ts` file.

|===

.Example remote module details
image::extend/add_application_manually/create_ui_component_remote_module.png[Example of filled remote module details form]

* Click `Save` to add the UI component for the application.

* Check that the UI component is added to the application created in the previous step. You can click on the application in the application store and click the `Components` tab to see the list of UI components added for the application. 

.Successfully added UI component
image::extend/add_application_manually/successfuly_added_ui_component.png[Example of the UI component added to the application]

[NOTE]
====
If you do not see the added UI component in the list, make sure to check that the `Application Name` in the management details of the UI component matches the `Name` field value of the application.
====
